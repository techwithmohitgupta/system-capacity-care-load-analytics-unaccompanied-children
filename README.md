## 📌 Project Overview

This project is an interactive **System Capacity & Care Load Analytics Dashboard** focused on monitoring the care pipeline for **unaccompanied children**.

The dashboard analyzes daily operational indicators across CBP custody, HHS care load, transfers into HHS care, discharges from HHS care, net intake pressure, backlog accumulation, care load volatility, and system pressure classification.

It is designed to help public-sector, healthcare operations, policy analytics, and capacity planning teams answer critical operational questions:

- What is the current total system load?
- How much of the care responsibility is concentrated in HHS care?
- Are discharges offsetting transfers into care?
- Is net intake pressure increasing or decreasing?
- Is backlog accumulating or easing?
- Which periods show high system strain?
- How volatile is daily care load movement?
- What pressure level should decision-makers monitor?
- Which operational areas require capacity planning attention?

---

## 🎯 Business Problem

Care systems involving unaccompanied children require continuous monitoring of intake, custody, care load, transfers, discharges, and backlog pressure.

Without a centralized analytics dashboard, it becomes difficult to monitor:

- Total system load
- CBP custody pressure
- HHS care responsibility
- Transfer and discharge flow
- Net intake pressure
- Backlog accumulation
- Care load volatility
- Pressure classification
- Operational stress windows
- Capacity planning requirements

This project solves that problem by converting daily care pipeline data into a dynamic Streamlit dashboard with KPIs, trend visuals, pressure indicators, operational summaries, and decision-ready recommendations.

## 🧩 Dashboard Modules

The dashboard is divided into four major analytical modules.

---

### 1️⃣ Executive Capacity Overview

This module summarizes the current care system status using decision-ready KPIs.

**Key Features**

- Total Children Under Care
- Net Intake Pressure
- Discharge Offset Ratio
- Care Load Volatility Index
- Backlog Accumulation Rate
- Overall KPI Health Score
- Peak System Load
- Current Load vs Peak
- Latest HHS care share
- Latest CBP custody share

**Business Value**

This section helps leadership quickly understand whether the system is operating under relief, pressure, or watch-list conditions.

---

### 2️⃣ Intake, Transfer & Discharge Flow Analysis

This module evaluates how children move through the system from custody to care and discharge.

**Key Features**

- Transfers into HHS care
- Discharges from HHS care
- Net intake total
- Daily intake pressure
- Transfer-to-discharge comparison
- Discharge offset ratio
- Flow imbalance indicators
- Relief-oriented vs pressure-oriented interpretation

**Business Value**

This section helps determine whether discharge throughput is strong enough to offset incoming transfers and reduce care load pressure.

---

### 3️⃣ Backlog & System Pressure Monitoring

This module focuses on backlog accumulation, care load growth, and operational pressure classification.

**Key Features**

- Backlog Accumulation Rate
- Rolling pressure windows
- Prolonged strain indicators
- Stress classification
- Daily system pressure movement
- Pressure severity comparison
- System load peaks and easing periods

**Business Value**

This section helps identify when care capacity pressure is accumulating and when operational intervention may be required.

---

### 4️⃣ Care Load Distribution & Capacity Planning

This module analyzes responsibility distribution between CBP custody and HHS care.

**Key Features**

- CBP custody share
- HHS care share
- Total system load movement
- Current load vs peak load
- Care load volatility
- System capacity utilization signals
- Downstream care responsibility concentration
- Capacity planning insights

**Business Value**

This section helps show where the operational burden is concentrated and supports planning around downstream shelter, care, and discharge capacity.

---

## 🧮 KPI Logic

```text
Total Children Under Care =
CBP Custody + HHS Care

Net Intake Pressure =
Transfers into HHS - Discharges from HHS Care

Discharge Offset Ratio =
Discharges from HHS Care / Transfers into HHS

Care Load Volatility Index =
Standard Deviation of Care Load Movement / Average Care Load × 100

Backlog Accumulation Rate =
Backlog Accumulation Periods / Total Observed Periods × 100

Current Load vs Peak =
Current Total System Load / Peak Total System Load × 100

Overall KPI Health =
Composite score based on intake balance, backlog pressure, care load stability, and system load position
```

## ▶️ How to Run Locally

### 1. Clone the Repository

```bash
git clone https://github.com/techwithmohitgupta/system-capacity-care-load-analytics-unaccompanied-children
```

### 2. Navigate to the Project Folder

```bash
cd system-capacity-care-load-analytics-unaccompanied-children
```

### 3. Create a Virtual Environment

```bash
python -m venv venv
```

### 4. Activate the Virtual Environment

For Windows:

```bash
venv\Scripts\activate
```

For macOS/Linux:

```bash
source venv/bin/activate
```

### 5. Install Dependencies

```bash
pip install -r requirements.txt
```

### 6. Run the Streamlit App

```bash
streamlit run app/app.py
```

## 🏁 Conclusion

This project demonstrates how daily care pipeline data can be converted into operational capacity intelligence for monitoring system load, care responsibility, transfer-discharge balance, backlog pressure, and care load volatility.

The dashboard helps move public-sector capacity monitoring from static reporting to dynamic, evidence-based decision support by identifying intake pressure, discharge relief, backlog accumulation, care load concentration, and pressure classification patterns.

By combining Python, Pandas, NumPy, Plotly, Streamlit, and published research documentation, this project provides an end-to-end applied data analytics portfolio case study for public-sector and healthcare capacity analytics.
